## 2026-07-10 (2)

- **1.2.0 phase 12b complete: optional semantic-rerank layer** (per `docs/1.2.0_PLAN.md`), built on Opus at High effort with a live adversarial pass on the never-blocks guarantee. New config key `ollama.embed_model` (default `None` -> feature disabled by construction; threaded through `configure_models`/`set_ollama`/the `set-ollama --embed-model` CLI flag). New `ollama_client.embed()`: bounded, non-blocking — `embed_model` unset returns `disabled` with zero network; a down/timed-out/model-missing Ollama is gated out by a `status()` probe before the POST; timeouts/exceptions return not-ok, never raise. Embeddings generate lazily inside `index_repo` for changed paths only, and only when `embed_model` is set — the loop breaks on the first non-ok embed so a down Ollama costs one probe, not one-per-file. Ranking blend (`_apply_semantic_rerank` in `mcp/memory.py`) re-ranks **only the already-FTS-selected candidates** by `cosine(task_vec, stored_file_vec) * 18` (sized below any structural signal, like the retrieval boost); it is gated on `repo_has_embeddings()` so an `embed_model`-unset repo never calls `embed()` and is byte-identical to pre-12b.
