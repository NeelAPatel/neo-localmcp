New CLI commands `neo-localmcp servers` (list) and `neo-localmcp stop` (`--pid`/`--all`/`--match-executable`/`--timeout`/`--no-force`), CLI-only by design so a remote MCP client cannot stop servers. `doctor` now reports `running_servers`.

Found and fixed a real Windows quirk during live verification: even after a process is confirmed exited, Windows can hold its DLL file lock for a moment longer (deferred handle release / AV scan-on-close), so both scripts now retry a removal briefly before treating it as a genuine lock.
